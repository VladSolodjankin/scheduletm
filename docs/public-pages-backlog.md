# Backlog: Public pages, account slug и кастомный лендинг

Ниже — согласованный список требований для документирования и дальнейшей реализации.

## 1) Account slug management

- Добавить в `account settings` поля:
  - `account_company_name` (редактируемое имя компании для публичной страницы);
  - `account_company_id` (уникальный slug/идентификатор аккаунта).
- Роли доступа: редактирование только для `owner` и `admin` аккаунта.
- При создании аккаунта генерировать `account_company_id` автоматически (slugify от названия + суффикс при конфликте), выбирая свободное значение.
- Добавить отдельный пункт меню в web-интерфейсе: **Public Page Settings** (внутри account-level настроек).

## 2) Public appointment status (без регистрации)

- Добавить публичную страницу проверки статуса записи для незарегистрированного пользователя.
- Входные параметры: `appointment_id + specialist_last_name`.
- Результат: отображение статуса appointment (например: `scheduled`, `rescheduled`, `cancelled`, `completed`) и базовых безопасных деталей без раскрытия лишних персональных данных.

## 3) Public booking by account slug

- Добавить публичный сценарий создания appointment без регистрации по ссылке формата:
  - `/{account_company_id}/login`
- Этот route должен открывать брендированную страницу аккаунта и позволять пройти сценарий записи как гость.

## 4) Настройки публичного лендинга (account-level)

Рекомендуется хранить в отдельной сущности (например, `account_public_page_settings`) и редактировать через отдельный раздел меню.

Обязательные настройки:
- Контакты и CTA-каналы:
  - Telegram;
  - Viber;
  - телефонный звонок;
  - кнопка «Записаться на услугу»;
  - опционально: WhatsApp, email, Instagram/соцсети.
- Сценарий записи на услугу:
  - отдельный flow при клике «Записаться на услугу»;
  - возможность выбора стандартного или кастомного сценария записи.
- Брендинг:
  - загрузка логотипа;
  - базовые фирменные цвета.
  - настройка `background` (изображение) или `background color` для страницы.
- Контент:
  - описание страницы/компании;
  - блок с картой и адресом.
- Кастомизация layout:
  - перенос (reorder) блоков;
  - создание/включение/выключение блоков.
- Расширенная визуальная кастомизация:
  - шрифты;
  - текстовые темы;
  - отступы (margin/padding);
  - размеры блоков.
- Responsive-first поведение:
  - корректное отображение на mobile/tablet/desktop обязательно.

## 5) Что еще добавить (рекомендации)

- SEO-настройки: `meta title`, `meta description`, OpenGraph image.
- Управление доменом: кастомный поддомен/домен для публичной страницы (в будущем).
- Аналитика конверсий: просмотры страницы, клики по CTA, завершенные записи.
- Шаблоны секций: быстрый старт с типовыми layout для разных типов бизнеса.
- Публикация по черновику: draft/preview/publish для безопасного редактирования.
- A/B-варианты hero-блока и CTA-кнопок.
- Anti-spam защита публичных форм (rate limit, captcha).
- Политики privacy/consent для публичной формы записи.

## 6) Дополнения по референсам конкурентов (из скриншотов)

На основе присланных референсов добавить в backlog следующие возможности для `Public Page Settings`:

### 6.1 Библиотека шаблонов и быстрый старт

- Каталог шаблонов с:
  - `Blank template` (создание с нуля);
  - готовыми шаблонами по категориям (например: Beauty, Health, Education, Law, Travel и т.д.).
- Поиск и фильтрация по категориям шаблонов.
- Отдельные вкладки/источники шаблонов:
  - Featured;
  - Landing pages;
  - My templates (пользовательские/сохраненные).

### 6.2 Theme presets (быстрая смена визуальной темы)

- Пресеты тем (палитра + типографика + стили кнопок/карточек) с быстрым переключением.
- Сценарий выбора:
  - предпросмотр пресета;
  - применить к странице;
  - вернуться к предыдущей версии (undo / version checkpoint).
- Поддержка редактирования пресета после применения (не только «фиксированный шаблон»).

### 6.3 Пошаговый flow создания публичной страницы

- В onboarding-конструкторе предусмотреть этапы:
  1. Choose template;
  2. Modify page;
  3. Get the link.
- На этапе `Get the link`:
  - показывать итоговый публичный URL;
  - кнопку копирования;
  - быстрые действия «открыть страницу», «поделиться».

### 6.4 Блочный редактор страницы

- Drag-and-drop reorder блоков.
- Кнопка `Add block` для добавления новых секций.
- Возможность:
  - скрывать/показывать блок;
  - удалять блок;
  - редактировать содержимое блока по клику.
- Минимальный набор типовых блоков:
  - Hero (фото/имя/подзаголовок/описание);
  - CTA links (звонок, мессенджеры, соцсети);
  - Service booking CTA;
  - Text/About;
  - Gallery;
  - Map/Address;
  - Contacts.

### 6.5 Preview / View режимы

- Отдельный `Preview` режим до публикации.
- Переключение Device preview: mobile / tablet / desktop.
- Гарантия WYSIWYG-поведения: то, что видим в preview, соответствует публичной странице.

### 6.6 CTA-блоки уровня «link in bio»

- Поддержать формат «список кнопок/ссылок», как в link-in-bio страницах:
  - позвонить;
  - Telegram;
  - WhatsApp/Viber;
  - Instagram/соцсети;
  - сохранить контакт (vCard);
  - записаться на услугу.
- Для каждой CTA-кнопки:
  - иконка;
  - заголовок;
  - при необходимости подзаголовок;
  - аналитика кликов.

### 6.7 Расширенный редактор отдельного блока (по вкладкам)

- Для каждого блока предусмотреть модальное окно редактирования с вкладками:
  - `Content`;
  - `Design`;
  - `Settings`;
  - `Section`.
- Вкладка `Content`:
  - выбор варианта layout блока (несколько пресетов);
  - загрузка/замена аватара или изображения;
  - редактирование headline/subtitle/текстовых полей.
- Вкладка `Design`:
  - настройка цвета текста;
  - font family;
  - размер/вес/стиль шрифта;
  - размер аватара/медиа-элементов.
- Вкладка `Settings`:
  - toggle `Hide block`;
  - вертикальные отступы;
  - опциональное отображаемое имя блока (служебный label).
- Вкладка `Section`:
  - привязка блока к секции страницы;
  - top/bottom padding;
  - edge indent;
  - section background color;
  - section background image (с fallback на color).

### 6.8 Библиотека блоков и повторное использование

- При `Add block` показывать каталог `Standard blocks` + `From archive`.
- Минимально предусмотреть типы блоков:
  - Text, Link, Divider, Avatar, FAQ;
  - Media and text, Messaging apps, Social networks, Map;
  - Video, Image, Image carousel;
  - HTML code (опционально, с ограничениями безопасности).
- Поддержать сценарий сохранения блока в архив и повторного использования.

### 6.9 Каталоги интеграций для social/video/music

- Для блока `Social networks` добавить встроенный каталог платформ:
  - social (например: Instagram, Telegram, TikTok, LinkedIn, VK, YouTube short links и т.д.);
  - video (YouTube, Vimeo и др.);
  - music (Spotify, Apple Music, SoundCloud и др.).
- Требования к UX:
  - визуально узнаваемые карточки платформ;
  - быстрый поиск;
  - возможность отключать неактуальные платформы на уровне аккаунта.

### 6.10 Тарифные ограничения и фичи (free/pro/biz)

- Явно зафиксировать поддержку feature flags для типа блока/опции:
  - доступно всем;
  - только `pro`;
  - только `biz`.
- Для недоступных фич:
  - отображать badge уровня (`pro`/`biz`);
  - понятный upgrade CTA;
  - не ломать текущий layout при недоступности опции.

### 6.11 Add-ons и встроенная статистика по блоку

- На уровне блока предусмотреть вкладки/разделы `Add-ons` и `Statistics`.
- `Add-ons`:
  - подключаемые расширения (пиксели, кастомные интеграции, виджеты — по whitelist).
- `Statistics`:
  - просмотры блока;
  - клики по CTA;
  - CTR по каждой кнопке/ссылке;
  - динамика по периоду.

### 6.12 Настройка нескольких профайлов (multi-profile)

- Добавить поддержку нескольких публичных профайлов внутри одного аккаунта (например: филиалы, направления услуг, отдельные специалисты).
- Для каждого профайла:
  - отдельный slug (например: `/{account_company_id}/{profile_slug}`);
  - собственный шаблон/тема/набор блоков;
  - собственные CTA и контакты;
  - собственное описание, логотип/обложка и background.
- Управление профайлами в `Public Page Settings`:
  - создать профайл;
  - дублировать существующий профайл;
  - архивировать/деактивировать профайл;
  - назначить профайл по умолчанию.
- Права доступа:
  - `owner/admin` могут управлять всеми профайлами аккаунта;
  - `specialist` (опционально) может редактировать только назначенный ему профайл.
- Booking-flow:
  - возможность привязать профиль к конкретным услугам/специалистам;
  - при переходе по профайлу показывать релевантные слоты и условия записи.
- Аналитика:
  - отдельные метрики по каждому профайлу (views, clicks, conversions);
  - агрегированный отчет по аккаунту + срез по профайлам.
