# MVP plan: запуск бота без оплаты

Цель: довести текущий backend Telegram-бот до состояния, когда можно принимать реальные записи без платежей и без веб-кабинета.

## Приоритет 1 — «бот принимает и сохраняет записи стабильно»

1. Привести время к `app_settings.timezone` (IANA) и убрать фиксированный UTC+3.
2. Добавить защиту от двойного бронирования слота (уникальный индекс/транзакция).
3. Добавить отмену записи пользователем (`status = cancelled`).
4. Добавить базовую команду сброса сессии (`/start`, `/reset`) и кнопку «Назад» хотя бы на 1 шаг.

Критерий готовности:
- пользователь может записаться, посмотреть запись, перенести и отменить;
- два пользователя не могут занять один и тот же слот;
- время в интерфейсе и БД предсказуемо для выбранной timezone.

## Приоритет 2 — «минимальная операционка»

1. Добавить напоминания в Telegram за 24 часа (канал только Telegram).
2. Включить управляемый `AUTO_SET_WEBHOOK` (в dev не трогать webhook).
3. Добавить `npm run typecheck` и запускать его в локальном CI/перед деплоем.

## Что можно отложить (не блокирует MVP без оплаты)

- BPMN-движок и BPMN-редактор.
- Мульти-аккаунт веб-кабинет и публичная регистрация пользователей.
- Email/SMS-уведомления.
- Полноценная ролевая модель и админка.

## Рекомендуемый первый технический шаг (сегодня)

Сделать один «вертикальный» инкремент:
1. timezone на IANA + корректные слоты,
2. блокировка двойной записи,
3. отмена записи.

Это даст рабочее ядро сервиса записи даже без оплаты и без web UI.
